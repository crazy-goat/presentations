Technology Overview

Technology Overview

Page composition can be categorized into four types based on where the assembly occurs: backend composition, server-side composition, edge-side composition, and client-side composition. Each approach determines where and how different components are combined to render the final webpage, impacting performance, flexibility, and caching strategies.

Technology Overview - backend composition

‚úÖ Pros:
  • Easy Implementation ‚Äì Simple to develop and maintain within a monolithic backend.
  • Strong Integration ‚Äì Direct access to databases and business logic.
‚ùå Cons:
  • Caching Challenges ‚Äì Difficult to cache dynamic content efficiently.
  • Performance Issues ‚Äì Most web languages are single-threaded, potentially causing bottlenecks.
  • Monolithic Architecture ‚Äì Encourages tightly coupled systems, reducing flexibility and scalability.

Technology Overview - client side

‚úÖ Pros:
  • Lazy Loading ‚Äì Loads only the necessary components, improving performance and user experience.
  • Frontend Rendering ‚Äì Moves HTML generation from server to client, reducing backend complexity.
  • Flexible and Scalable ‚Äì Works well with microservices and APIs, allowing independent frontend updates.
‚ùå Cons:
  • Slower Initial Load ‚Äì More processing happens in the browser, which may delay rendering.
  • SEO Challenges ‚Äì Search engines may struggle to index client-rendered content without proper server-side support.
  • Inconsistent Performance ‚Äì Heavily depends on the user‚Äôs device and network speed.

Server Side Include (SSI)

Server Side Include (SSI)

Server Side Includes (SSI) is a server-side feature that allows web pages to include content from other sources before being sent to the client. It works by processing special SSI directives, such as:

which enable the inclusion of external files or subrequests. It is supported by Apache, LiteSpeed, Nginx and IIS servers

Apache SSI

In Apache, configuring Server Side Includes (SSI) is straightforward because the mod_include module is compiled by default and only needs to be activated.

Enable: 
a2enmod include
Config: 
Options +Includes
AddOutputFilter INCLUDES .php .shtml .html

Nginx SSI

In Nginx, configuring Server Side Includes (SSI) can be more complex because the ngx_http_ssi_module is not compiled by default on some distributions and may need to be enabled during the build process.

Enable: 
./configure --with-http_ssi_module
Config: 
location / {
    ssi on;
}

Technology Overview - server side composition

‚úÖ Pros:
  • Improved Performance ‚Äì Enables partial page caching, reducing backend load.
  • Efficient Content Assembly ‚Äì Components are fetched in parallel, speeding up page rendering.
  • Multi-Language Support ‚Äì Components can be written in different programming languages and integrated seamlessly.
‚ùå Cons:
  • Limited Flexibility ‚Äì Requires server-side support and specific configurations.
  • Increased Complexity ‚Äì Managing dependencies between fragments can be challenging.
  • Security Risks ‚Äì Incorrect configuration may introduce vulnerabilities, such as exposing internal services or enabling injection attacks.

Technology Overview - edge side composition

‚úÖ Pros:
  • Reduced Backend Load ‚Äì Pages are partially assembled at the CDN level, reducing server strain.
  • Faster Content Delivery ‚Äì Components can be cached closer to the user, improving response times.
  • Easy to Configure ‚Äì Many CDNs offer built-in ESI support with minimal setup required.
‚ùå Cons:
  • Limited Personalization ‚Äì Cached content may not adapt well to user-specific data.
  • Complex Debugging ‚Äì Issues may be harder to trace due to processing happening at the edge.
  • CDN Dependency ‚Äì Requires a CDN with ESI support, limiting flexibility in infrastructure choices.
  • Additional Costs ‚Äì CDN services with ESI support can come with extra fees.
